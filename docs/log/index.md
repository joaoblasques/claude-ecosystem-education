# The learning log

This is the public record — Learner #0's daily log of going through the program while
building it. Every participant keeps their own version of this on their fork.

## The format

One page per working day, named `YYYY-MM-DD.md`:

```markdown
# YYYY-MM-DD — Day N (Level X, Day/Module Y)

**Shipped:** <the artifact, with a link — or "nothing shipped" and why>

## What I did
2–5 bullets. Facts, not vibes.

## What broke / surprised me
The most useful section for everyone else. UI changed? Model refused?
Docs wrong? Write it down.

## Tomorrow
One line.
```

## Rules

- Write it the same day, in under five minutes. Perfect is the enemy of posted.
- Never log work that didn't happen. Gaps are honest; padding isn't.
- Link artifacts. A log that says "built a thing" without a link didn't ship.
