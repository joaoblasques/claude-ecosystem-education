# Day 1 — Meet the Ecosystem

**Goal:** Configure claude.ai properly — Projects, Memory, Extended Thinking, web search, document upload — and map all five categories of what Claude can actually do.

## Why this matters

Most people use Claude the way they used a search box: type a question, read the answer, close the tab. That leaves the majority of the product on the table. The features that make Claude a working tool rather than a novelty — persistent Projects, Memory, step-by-step reasoning, live web search — are all one or two clicks away, and almost nobody turns them on. Today you turn them on and take a structured tour of the territory the rest of the week will cover.

## Concepts

### The user → operator shift

There is a difference between *asking Claude things* and *operating Claude*: building repeatable workflows where context persists, output formats are specified, and the same task gets better every time you run it. Everything in Level 1 pushes you across that line.

### The five capability categories

A useful mental map organizes Claude's ~25 usage modes into five buckets:

- **Think** — Extended Thinking, deep research with web search, interrogating uploaded documents.
- **Create** — long-form drafts inside Projects, prompt building, Claude Design, presentations.
- **Build** — Artifacts (Day 2), and later Claude Code (Level 2).
- **Connect** — Connectors/MCP (Day 4), web search, Chrome and Excel integrations (Day 5).
- **Automate** — Skills (Day 3), Cowork, and loops (Day 7).

You don't need all 25 today. You need to know the map exists so nothing this week feels like an isolated trick.

### The persistence layer: Projects and Memory

- **Projects** (sidebar → new Project, as of mid-2026) are folders that hold instructions and reference documents Claude reads in every conversation inside them. A Project with a style guide beats re-pasting your preferences forever.
- **Memory** (Settings → Memory) lets Claude retain facts about you across all conversations. You can view and edit what it has stored — do that today; treat it like any other config file.

### The reasoning and evidence layer

- **Extended Thinking** (toggle in the model selector) makes Claude work stepwise before answering instead of producing a confident one-shot reply. Use it for anything with real stakes or multiple constraints.
- **Web search** (the "+" menu in the chat bar) grounds answers in current, cited sources. Default it on for anything time-sensitive.
- **Document upload** ("+" → upload) works far better with *specific* questions ("what are the three weakest assumptions in section 4?") than with "summarize this."

!!! warning "Specificity is the whole game"
    A weak brief produces a weak draft, no matter which features are enabled. Angle, audience, goal, format, length — say them every time. This rule repeats on every day of Level 1.

### What deliberately waits for later days

Three modes from the map are worth naming now so you don't chase them today: Artifacts (Day 2), Skills (Day 3), and Connectors (Day 4) each get a full day. Claude Code — the terminal tool the "Build" category ultimately points at — is Level 2 material entirely. Today is configuration and cartography, nothing more.

## Build — ship something today

Build your **Level 1 home base** and a **personal scorecard**.

1. Create a Project named `Level 1 — Claude Apps`. In its instructions, write 5–10 lines about who you are, what you do, and how you like output formatted. This Project hosts every Build this week.
2. In Settings → Memory (as of mid-2026), read what Claude has stored about you. Edit or add at least one fact.
3. Enable Extended Thinking and ask one genuinely hard question from your actual work. Compare against the same question with it off.
4. Upload one real PDF (a report, a contract, a paper) and ask three *specific* questions about it.
5. In the Project, create a scorecard note: list the five categories (Think/Create/Build/Connect/Automate) and under each, write which modes you already use, which you'll adopt this week, and which you're skipping — with one line of reasoning each.

Ship: the scorecard, posted publicly (screenshot or text).
